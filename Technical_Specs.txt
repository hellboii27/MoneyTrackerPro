TECHNICAL SPECIFICATION DOCUMENTATION - MONEY TRACKER PRO
================================================================================
Project Name     : Money Tracker Pro
System Version   : v4.2.0-Stable
Base Environment : Google Apps Script (GAS) V8 Runtime
Database         : Flat-File Relational (Google Sheets API)
================================================================================

1. ARCHITECTURAL OVERVIEW
--------------------------------------------------------------------------------
Money Tracker Pro adopts an N-Tier Thin Client architecture. All transactional 
logic (CRUD) and balance synchronization are processed on the server-side (GAS), 
while visual rendering and UI state are managed entirely on the client-side 
using ES6+ standards.

2. FRONTEND SPECIFICATIONS
--------------------------------------------------------------------------------
- UI Framework    : Bootstrap 5.3.0 (Responsive Container 480px).
- Typography      : Plus Jakarta Sans (Weight 400-800).
- Charting Engine : Chart.js 4.4.1 (Doughnut Object).
- Interaction Design (Slide-to-Action):
    * Technology  : JavaScript Touch Events (touchstart, touchmove, touchend).
    * Logic       : Touch-based horizontal translation (X-axis).
    * Safety Layer: Dynamic implementation of 'visibility' and 'opacity' to 
                    prevent visual background leakage on rounded container 
                    corners when the component is in an idle position (0px).
    * Timing      : 0.25s cubic-bezier spring effect for smooth transitions.

3. DATA MANAGEMENT & DATABASE SCHEMA
--------------------------------------------------------------------------------
The system operates using two primary entities via the Google Sheets API:
- Entity 'Transaksi': [Tanggal, Tipe, Kategori, Wallet, Nominal, Catatan]
- Entity 'Wallets'  : [Nama Wallet, Saldo]

Data Integrity:
- Input Sanitization: Stripping non-numeric characters via regex before 
  database injection.
- State Sync: Utilizing 'withSuccessHandler' to ensure data integrity between 
  the UI and server-side database.

4. EXPORT PROTOCOL
--------------------------------------------------------------------------------
- Output Format   : CSV (RFC 4180 Compliant).
- Encoding        : UTF-8 with Byte Order Mark (BOM) for optimal compatibility 
                    with Microsoft Excel.
- Data Processing : Client-side Blob generation for server resource efficiency.
